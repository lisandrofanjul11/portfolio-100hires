# Cursor Setup & Configuration Test

## Tools Installed
* **Cursor**: The main code editor I used for this test.
* **Claude Code for VS Code**: Official Anthropic extension.
* **Codex**: Coding agent extension.
* **Git**: For version control on my Mac.

## Steps Completed
1. Downloaded and installed Cursor on my Mac.
2. Located the traditional VS Code-style extensions panel.
3. Installed Claude Code and Codex, and logged into both tools.
4. Logged into my GitHub account and created a new public repository named `portfolio-100hires`. I made sure to leave it completely empty (no default README).
5. Cloned the repository to my computer using the `Git: Clone` command inside Cursor.
6. Wrote this `README.md` file manually to document the whole process.

## Issues Encountered & How I Solved Them
* **Issue 1: Navigating the new Cursor interface.** When I first opened the program, it took me to a "Composer" screen and an internal Plugins menu, so I couldn't find the regular extensions store.
  * **Solution:** I used the Command Palette to open the "Editor Window". This switched the app to the classic view where I could easily find the correct sidebar to search for the extensions.
* **Issue 2: Mac prompted for extra tools.** When I tried to use Git to clone the repo, macOS threw an error because the Apple command line developer tools were missing.
  * **Solution:** I accepted the prompt to download the tools. It took about 15 minutes to install. After that, I fully restarted Cursor so it could recognize the new Git commands, which let me clone the folder without any issues.
